# Splash Screen

## Overview

Consider whether to add a splash screen on app launch. This is a "maybe not" — the question is whether it adds value or just gets in the way.

## Questions to Answer

- [ ] Do I want a splash screen at all?
- [ ] What would it show? Branding? Tips? Recent hypnograms?
- [ ] Would it delay getting to the actual app experience?
- [ ] Is this solving a real problem or just adding polish?

## Arguments Against

- Hypnograph is a creative tool — users want to get into the flow, not wait
- macOS apps generally don't have splash screens (it's more of a Windows/mobile pattern)
- The app loads fast enough that a splash would feel artificial

## Arguments For

- Could show the quote: "What you are looking for is who is looking." - St. Francis of Assisi
- First-run onboarding could live here (permissions, source folder setup)
- Branding moment for a distinctive app identity

## Decision

Canceled on 2026-02-17 — not shipping a splash screen unless a concrete first-run onboarding need emerges.
